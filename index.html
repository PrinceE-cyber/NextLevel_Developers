Week 1

HTML Fundamentals — Part 1

3 Lessons  ·  Days 1 – 6  ·  HTML Only

🎯 Objective

Understand what HTML is, why it exists, and how to write a proper HTML document from scratch using the correct boilerplate structure.

Day 1: Learn the concepts. Write and run the boilerplate in your browser.  |  Day 2: Practise independently. Build your own bare-bones webpage skeleton.

📖 Explanation

HTML stands for HyperText Markup Language. It is the standard language used to create and structure content on the web. Think of HTML as the skeleton of a web page — it gives everything its shape and meaning, but no colour or style yet. That comes later with CSS.

HTML is made up of elements. Each element is written using tags. Most tags come in pairs: an opening tag <tag> and a closing tag </tag>, with content in between. Some tags are self-closing, like <br /> and <img />.

Every HTML document must start with a specific boilerplate — a standard set of lines that tells the browser what kind of document it is reading and how to display it. You will type this boilerplate in every project you ever build.

The four key parts of the boilerplate are:

  1. <!DOCTYPE html> — tells the browser this is a modern HTML5 document.
  2. <html> — the root element that wraps everything.
  3. <head> — contains invisible information about the page (title, character set, linked files).
  4. <body> — contains everything the user actually sees on the page.

💻 Code Example

<!DOCTYPE html>
<html lang="en">

  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My First Page</title>
  </head>

  <body>
    <h1>Hello, World!</h1>
    <p>This is my very first web page.</p>
  </body>

</html>

Save this as index.html, open it in Chrome or Firefox, and you will see "Hello, World!" displayed as a large heading. Every project from now on starts with this exact structure.

✏️ Practice Task — Day 1

  1. Open VS Code. Create a new folder called week1 on your desktop.
  2. Inside it, create a file called index.html.
  3. Type (never copy-paste!) the full boilerplate above from memory.
  4. Change the <title> text to your own name.
  5. Change the <h1> to say your name and a greeting.
  6. Add a <p> below it with one sentence describing yourself.
  7. Open the file in your browser — you just built a web page. 🎉

✏️ Practice Task — Day 2

  1. Close your editor and delete what you wrote.
  2. Re-type the boilerplate from memory — no peeking at notes.
  3. Add an <h1>, an <h2>, and three <p> tags with any content you like.
  4. Open it in the browser and check that it looks correct.
  5. Repeat until you can write the boilerplate without any help. This is muscle memory you will use forever.

🎯 Objective

Learn how to format text using heading tags (h1–h6), paragraphs, bold, italic, and how to build both ordered and unordered lists.

Day 3: Study each tag and type every example in your editor.  |  Day 4: Build an "About Me" page using only what you learned today — no looking at the lesson.

📖 Explanation

HTML gives you six heading levels: <h1> is the biggest and most important. <h6> is the smallest. Use only one <h1> per page — it tells the browser (and search engines) what the page is mainly about.

For normal body text, use the <p> (paragraph) tag. Each <p> automatically adds space above and below it.

To add emphasis inside text, use <strong> for bold/important content and <em> for italic/emphasis. These are semantic tags — they carry meaning, not just visual style.

Lists come in two types. An unordered list <ul> creates bullet points. An ordered list <ol> creates numbered steps. Each item in both types uses the <li> tag.

💻 Code Example

<body>

  <!-- Headings: h1 is the biggest, h6 the smallest -->
  <h1>My Name Is Edison</h1>
  <h2>About Me</h2>
  <h3>My Hobbies</h3>

  <!-- Paragraph with emphasis -->
  <p>
    I am learning <strong>HTML</strong> and I find it
    <em>very exciting</em>.
  </p>

  <!-- Unordered list = bullet points -->
  <h3>My Favourite Foods</h3>
  <ul>
    <li>Rice and Cassava Leaf</li>
    <li>Bread</li>
    <li>Groundnut Soup</li>
  </ul>

  <!-- Ordered list = numbered steps -->
  <h3>How I Start My Day</h3>
  <ol>
    <li>Wake up at 6am</li>
    <li>Exercise for 20 minutes</li>
    <li>Open VS Code and code</li>
  </ol>

</body>

✏️ Practice Task — Day 3

  1. Open your index.html from Days 1–2.
  2. Add an <h2> that says "My Favourite Foods".
  3. Below it, create a <ul> with at least 4 food items.
  4. Add an <h2> that says "My Daily Routine".
  5. Create an <ol> with at least 5 numbered steps.
  6. Use <strong> to bold at least one important word in a paragraph.
  7. Use <em> to italicise at least one word.

✏️ Practice Task — Day 4

  1. Create a brand new file: about.html.
  2. Write the full boilerplate from memory.
  3. Build a complete "About Me" page with: your name as <h1>, at least 3 sections with <h2> headings, at least 2 paragraphs, one unordered list, and one ordered list.
  4. Use <strong> and <em> at least once each.
  5. Open in browser and make sure everything displays correctly.

🎯 Objective

Learn how to add images to a webpage using <img> and how to create clickable links using <a> — both to other websites and between your own pages.

Day 5: Study the img and anchor tags. Add images and links to your existing pages.  |  Day 6: Build a hobby page with images and links. Connect it to your About page.

📖 Explanation

The <img> tag is used to display an image. It is a self-closing tag — it has no closing tag. It requires two important attributes:

  1. src — the path to the image file (can be a local file or a web URL).
  2. alt — a text description of the image for screen readers and when the image fails to load. Always include this.

The <a> tag creates a hyperlink. The text between the tags is what users click. The href attribute tells the browser where to go. It can point to another website (absolute URL) or another page in your own project (relative path like about.html).

To open a link in a new browser tab, add the attribute target="_blank". Always pair it with rel="noopener" for security.

💻 Code Example

<body>

  <h1>My Hobbies</h1>

  <!-- Image from the web (absolute URL) -->
  <img
    src="https://upload.wikimedia.org/wikipedia/commons/thumb/6/6d/Good_Food_Display_-_NCI_Visuals_Online.jpg/640px-Good_Food_Display_-_NCI_Visuals_Online.jpg"
    alt="A display of healthy foods"
    width="400"
  />

  <!-- Image from a local file (relative path) -->
  <img src="images/my-photo.jpg" alt="A photo of me" />

  <!-- Link to another website (opens in new tab) -->
  <a href="https://www.google.com" target="_blank" rel="noopener">
    Visit Google
  </a>

  <!-- Link to another page in your own project -->
  <a href="about.html">Go to my About page</a>

  <!-- Link that jumps to a section on the same page -->
  <a href="#hobbies">Jump to Hobbies section</a>

  <!-- The section being jumped to must have a matching id -->
  <h2 id="hobbies">My Hobbies</h2>

</body>

✏️ Practice Task — Day 5

  1. Open your about.html from Day 4.
  2. Add at least 2 images using web URLs (search Google Images, right-click → Copy image address).
  3. Make sure every image has a descriptive alt attribute.
  4. Add a link to your favourite website, opening in a new tab.
  5. Add a link at the top of the page that jumps down to a section further below using an id.

✏️ Practice Task — Day 6 (Mini Project)

  1. Create a new file: hobbies.html.
  2. Write the boilerplate from memory.
  3. Build a Hobbies page with: a <h1> title, at least 3 hobbies each with an <h2>, a paragraph, and an image.
  4. At the bottom of hobbies.html, add a link back to about.html.
  5. On about.html, add a link to hobbies.html.
  6. You now have two connected pages — a mini website! Test both links in your browser.
